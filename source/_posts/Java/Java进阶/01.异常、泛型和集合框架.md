## 一、异常
> 异常是程序运行时出现的错误，导致程序无法继续运行的情况。

### 异常的分类
* 运行时异常（RuntimeException）
  编译阶段不会出现错误提醒，运行时出现的异常(如:数组索引越界异常)编译时异常:
**注意**：运行时异常很可能会导致程序中断！
> 例如大家熟悉的空指针异常基本都是运行时异常。
* 编译时异常（Exception）
  编译阶段就会出现错误提醒的，且编译阶段不可通过。(如:日期解析异常)
> 出现代码编辑器的错误提示基本都是编译时异常。主要为了提醒编码者这里代码有问题。

### 异常的处理
1. 抛出异常：又叫找背锅侠，就是把异常抛给调用者处理。
2. 捕获异常：自己捕获异常，自己处理。
```java
/**
 * 异常的处理样例
 *
 * Version: 1.0
 * Author: 瓶子
 */
public class ExceptionDemo {
    public static void main(String[] args) {
        try {                   // 尝试捕获异常
          show();               // 未出现异常正常执行。
        }
        catch (java.lang.Exception e) {      // 出现异常，捕获异常会执行下面的代码块
          System.out.println("字符串为NULL");;
        }
        
        try{
          System.out.println(show2(0));
        }
        catch (Exception e){
            e.printStackTrace();
            System.out.println("程序出现异常");
        }
    }
    public static void show() throws Exception {  //throws抛出异常，Exception是抛出的的异常类，这里也可以同时抛出多个异常类。
        String s = null;
        s.length();
    }    
    
    public static int show2(int a) throws Exception {       // 另一种抛出异常的方式（推荐），手动抛出异常提示便于在日志重定位异常代码位置。方便修复bug
        if (a = 0){
            // 手动返回异常给上层，还能告知程序是否执行成功。
            throw new Exception("除数不能为0");
        }
        int result = 10/a;
        return result;
    }
    
}
```

### 异常的作用
1、用来定位程序BUG的关键信息。
> 例如上面的例子，当字符串为NULL时，会抛出Exception异常，并执行catch块中的代码。

2、作为方法的特殊返回，以便通知上层调用者需要进行什么处理。
> 当方法执行成功时，返回一个结果，当方法执行失败时，返回一个异常。便于定位BUG

## 二、泛型
### 认识泛型
>定义类、接口、方法时，同时声明了一个或者多个类型变量（如`<E>`）
>称为泛型类，泛型接口，泛型方法，统称泛型。

作用：泛型提供了在编译阶段约束集合元素类型的能力，并进行自动检查的能力。可以避免强制类型转换，及可能出现的异常。
本质：泛型本质上是把具体的数据类型作为参数传递给类型变量。

### 泛型类
> 声明时定义了泛型的类就是泛型类，声明类型变量时，类型变量名称必须大写。
```java
// 修饰符 class 类名<类型变量> { ... }
public class GenericClass<T>{  }  // 泛型类
```

### 泛型接口
> 声明接口时时定义了泛型的接口就是泛型接口，声明类型变量时，类型变量名称必须大写。
```java
// 修饰符 interface 类名<类型变量> { ... }
public interface Genericinterface<T>{  }  // 泛型接口
```

### 泛型方法
> 声明方法时时定义了泛型的方法就是泛型方法，声明类型变量时，类型变量名称必须大写。
```java
// 修饰符 <类型变量> 返回值类型 方法名(形参列表) { ... }
public class Generic{ 
    public <T> void show(T t){} // 泛型方法。
    public <T> int show(T t){}  // 泛型方法。
    public E show(int index){}  // 不是泛型方法
}  
```

### 泛型样例
```java
public class GenericDemo{
    public static void main(String[] args) { 
        // 未增加泛型限制的集合。集合中可以存放任意类型的数据
        ArrayList list1 = new ArrayList<>();
        list.add("hello");
        list.add(1);
        list.add(99.99);
        // 增加泛型限制的集合，集合中只能存放String类型数据
        ArrayList<String> list2 = new ArrayList<>();
        list2.add("hello");
        list2.add(1);   // 报错无法添加
    }
}
```

### 泛型通配符与上下限。
> 泛型通配符：`?`，可以在“使用泛型”的时候表示一切泛型；ETKV是在定义泛型的时候使用。

通配符一般配合上下限一起使用
> 泛型通配符的上限：`<? extends T>`，表示该泛型只能是T类或者T的子类；
> 泛型通配符的下限：`<? super T>`，表示该泛型只能是T类或者T的父类。
```java
/**
 * 泛型通配符和上下限展示
 *
 * Version: 1.0
 * Author: 瓶子
 */
public class Demo { 
    public static void main(String[] args) {
        ArrayList<Xiaomi> xiaomi = new ArrayList<>();
        xiaomi.add(new Xiaomi());
        ArrayList<Bus> bus = new ArrayList<>();
        bus.add(new Bus());
        
        // run(xiaomi);    // 报错，无法传入
        // run(bus);       // 报错，无法传入
        go(xiaomi);        // 不报错，正常运行
        go(bus);           // 不报错，正常运行
        ArrayList<Dog> dog = new ArrayList<>();
        dog.add(new Dog());
        go(dog);           // 不报错，正常运行，通配符导致方法传入了不该出现的参数。
        raceTrack(xiaomi); // 不报错，正常运行
        raceTrack(bus);    // 不报错，正常运行
        raceTrack(dog);    // 报错，无法传入。方法限制只能Car和其子类进入，Dog不是Car的子类。
        
    }
    // 虽然Xiaomi和Bus都是Car的子类，但是ArrayList<Xiaomi>、ArrayList<Bus>和ArrayList<Car> 之间没有任何关系，不能像
    // 继承那样直接使用父类有关的参数和方法。
    public static void run(ArrayList<Car> cars) {}
    // 使用通配符，可以传入一切类型变量。
    public static void go(ArrayList<?> cars) {}
    // 使用通配符和上下限，限制只有Car的子类可以进入方法
    public static void raceTrack(ArrayList<? extends Car> cars) {}
}
class Car {}
class Bus extends Car {}
class Xiaomi extends Car {}
class Dog{}
```

### 泛型支持的类型
> 泛型只支持对象类型（引用类型），不支持基本数据类型。

泛型如果需要使用基本数据类型，必须在声明泛型时使用基本数据类型的包装类。也就是被包装为对象类型的基本数据。
包装类对应关系如下：

| 基本数据类型 | 对应的包装类 |
| ----------- | ----------- |
| byte | Byte |
| short | Short |
| int | Integer |
| long | Long |
| float | Float |
| double | Double |
| char | Character |
| boolean | Boolean |

使用包装类的集合样例。
```java
public class Demo {
    public static void main( String[] args ) {
        // ArrayList<int> list = new ArrayList<>();        // 报错。
        ArrayList<Integer> list = new ArrayList<>();    // 正确
        list.add( 1 );                // 使用正确的方式添加元素。
        int number = list.get( 1 );   // 使用正确方式获取元素。
    }
}
```

## 三、集合框架
### 认识集合
> 集合是一种容器，用来装数据，类似数组。但是集合的大小可变，比数组更加灵活。

集合分两类，一类是`Collection`，一类是`Map`
* collection：又叫单列集合，集合中每个元素只包含一个值。
* Map：又叫双列集合，集合中每个元素包含两个值，一个键，一个值（键值对）。

### Collection集合
#### Collection集合特点
`Collection`集合下还有两种类别
* `List`系列集合：添加的元素是**有序的、可重复的**。
* * `ArrayList`、`LinekdList`：集合内容有序，可重复，有索引。

* `Set`系列集合：添加的元素是**无序的、不可重复的**。
* * `HashSet`：无序、不重复、无索引
* * `TreeSet`：按照数据大小默认升序排序，不重复，无索引。
* * `LinkedHashSet`：有序、不重复、无索引
* 这三种集合都是`Set`的子接口，他们的特点都是：无序，不可重复。但是每个都有和其他不一样的地方。


#### Collection集合特点展示
```java
/**
 * Collection集合特点展示
 *
 * Version: 1.0
 * Author: 瓶子
 */
public class CollectionDemo {
  public static void main(String[] args) { 
      // List集合展示：有序，可重复，有索引
      List<String> list = new ArrayList<>();
      list.add("hello");
      list.add("world");
      list.add("Java");
      list.add("Java");
      System.out.println(list);     // 输出：[hello, world, Java, Java]    顺序和添加顺序一致。
      String s = list.get(1);       // 输出：world   可以使用索引获取元素。
    
      // Set集合展示：无序，不可重复，无索引
      Set<String> set = new HashSet<>();
      set.add("hello");
      set.add("world");
      set.add("Java");
      set.add("Java");
      System.out.println(set);      // 输出：[Java, hello, world]   无序，不可重复。
      //set.get(1);                 // 不可用，没有get方法
  }
}
```

#### Collection集合的常用功能
Collection集合通用功能

| 方法名                                 | 描述                 |
|:------------------------------------|:-------------------|
| public boolean add(Ee)              | 把给定的对象添加到当前集合中     |
| public void clear()                 | 清空集合中所有的元素         |
| public boolean remove(Ee)           | 把给定的对象在当前集合中删除     |
| public boolean contains(Object obj) | 判断当前集合中是否包含给定的对象   |
| public boolean isEmpty()            | 判断当前集合是否为空         |
| public int size()                   | 返回当前集合中元素的数量       |
| public object[] toArray()           | 返回一个包含当前集合中所有元素的数组 |

#### Collection集合的遍历方式
##### 迭代器遍历
* 迭代器是用来遍历集合的**专用方式**（数组没有迭代器），java迭代器的代表是`Iterator`

Iterator迭代器常用方法

| 方法名                  | 方法功能                     |
| ----------------------- | ---------------------------- |
| public boolean hasNext() | 判断集合中是否还有下一个元素 |
| public Object next() | 返回集合的下一个元素 |

##### 增强for循环
格式
```textmate
for(元素的数据类型 变量名 : 集合或数组){ ... }
```
* 增强for可以用来遍历集合或数组
* 增强for循环遍历集合时，底层会调用集合对象的迭代器对象，通过迭代器对象来遍历集合元素。本质就是迭代器的简化写法。

##### Lambda表达式
> 此特性从JDK 8 开始支持。

此种方法需要使用 `Collection`的如下方法
|方法名|描述|
|:-----|:-----|
|default void forEach(Consumer<? super E> action)|对集合中的元素进行迭代，并执行指定的操作|

#### 三种遍历方式的区别


```java
/**
 * Collection集合的遍历方式演示
 * 
 * Version: 1.0
 * Author: 瓶子
 */
public class CollectionTraversalDemo {
  public static void main(String[] args) {
      Collection<String> carname = new ArrayList<>();
      carname.add("保时捷");
      carname.add("法拉利");
      carname.add("兰博基尼");
      System.out.println(carname);    // [保时捷, 法拉利, 兰博基尼]
    
      //1、迭代器--得到这个集合的迭代器对象
      Iterator<String> iterator = carname.iterator();    // 调用迭代器方法
      while (iterator.hasNext()) {          // 判断集合中是否有下一个元素
          String next = iterator.next();    // 获取集合中的下一个元素
          System.out.println(next);         // 输出元素
      }
      
      //2、增强for循环--得到集合中的元素
      for (String s : carname) {
          System.out.println(s);
      }
      
      //3、Lambda表达式遍历
      // 匿名函数，Consumer是一个接口式函数，可简化
//      carname.forEach(new Consumer< String >() {
//          @Override
//          public void accept( String s ) {
//              System.out.println(s);
//          }
//      });
      // 简化后。
      carname.forEach(s -> System.out.println(s));
  }
}
```
