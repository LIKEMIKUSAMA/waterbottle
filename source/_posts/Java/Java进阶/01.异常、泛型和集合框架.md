## 一、异常
> 异常是程序运行时出现的错误，导致程序无法继续运行的情况。

### 异常的分类
* 运行时异常（RuntimeException）
  编译阶段不会出现错误提醒，运行时出现的异常(如:数组索引越界异常)编译时异常:
**注意**：运行时异常很可能会导致程序中断！
> 例如大家熟悉的空指针异常基本都是运行时异常。
* 编译时异常（Exception）
  编译阶段就会出现错误提醒的，且编译阶段不可通过。(如:日期解析异常)
> 出现代码编辑器的错误提示基本都是编译时异常。主要为了提醒编码者这里代码有问题。

### 异常的处理
1. 抛出异常：又叫找背锅侠，就是把异常抛给调用者处理。
2. 捕获异常：自己捕获异常，自己处理。
```java
/**
 * 异常的处理样例
 *
 * Version: 1.0
 * Author: 瓶子
 */
public class ExceptionDemo {
    public static void main(String[] args) {
        try {                   // 尝试捕获异常
          show();               // 未出现异常正常执行。
        }
        catch (java.lang.Exception e) {      // 出现异常，捕获异常会执行下面的代码块
          System.out.println("字符串为NULL");;
        }
        
        try{
          System.out.println(show2(0));
        }
        catch (Exception e){
            e.printStackTrace();
            System.out.println("程序出现异常");
        }
    }
    public static void show() throws Exception {  //throws抛出异常，Exception是抛出的的异常类，这里也可以同时抛出多个异常类。
        String s = null;
        s.length();
    }    
    
    public static int show2(int a) throws Exception {       // 另一种抛出异常的方式（推荐），手动抛出异常提示便于在日志重定位异常代码位置。方便修复bug
        if (a = 0){
            // 手动返回异常给上层，还能告知程序是否执行成功。
            throw new Exception("除数不能为0");
        }
        int result = 10/a;
        return result;
    }
    
}
```

### 异常的作用
1、用来定位程序BUG的关键信息。
> 例如上面的例子，当字符串为NULL时，会抛出Exception异常，并执行catch块中的代码。

2、作为方法的特殊返回，以便通知上层调用者需要进行什么处理。
> 当方法执行成功时，返回一个结果，当方法执行失败时，返回一个异常。便于定位BUG

## 二、泛型
### 认识泛型
>定义类、接口、方法时，同时声明了一个或者多个类型变量（如`<E>`）
>称为泛型类，泛型接口，泛型方法，统称泛型。

作用：泛型提供了在编译阶段约束集合元素类型的能力，并进行自动检查的能力。可以避免强制类型转换，及可能出现的异常。
本质：泛型本质上是把具体的数据类型作为参数传递给类型变量。

### 泛型类
> 声明时定义了泛型的类就是泛型类，声明类型变量时，类型变量名称必须大写。
```java
// 修饰符 class 类名<类型变量> { ... }
public class GenericClass<T>{  }  // 泛型类
```

### 泛型接口
> 声明接口时时定义了泛型的接口就是泛型接口，声明类型变量时，类型变量名称必须大写。
```java
// 修饰符 interface 类名<类型变量> { ... }
public interface Genericinterface<T>{  }  // 泛型接口
```

### 泛型方法
> 声明方法时时定义了泛型的方法就是泛型方法，声明类型变量时，类型变量名称必须大写。
```java
// 修饰符 <类型变量> 返回值类型 方法名(形参列表) { ... }
public class Generic{ 
    public <T> void show(T t){} // 泛型方法。
    public <T> int show(T t){}  // 泛型方法。
    public E show(int index){}  // 不是泛型方法
}  
```

### 泛型样例
```java
public class GenericDemo{
    public static void main(String[] args) { 
        // 未增加泛型限制的集合。集合中可以存放任意类型的数据
        ArrayList list1 = new ArrayList<>();
        list.add("hello");
        list.add(1);
        list.add(99.99);
        // 增加泛型限制的集合，集合中只能存放String类型数据
        ArrayList<String> list2 = new ArrayList<>();
        list2.add("hello");
        list2.add(1);   // 报错无法添加
    }
}
```

### 泛型通配符与上下限。
> 泛型通配符：`?`，可以在“使用泛型”的时候表示一切泛型；ETKV是在定义泛型的时候使用。

通配符一般配合上下限一起使用
> 泛型通配符的上限：`<? extends T>`，表示该泛型只能是T类或者T的子类；
> 泛型通配符的下限：`<? super T>`，表示该泛型只能是T类或者T的父类。
```java
/**
 * 泛型通配符和上下限展示
 *
 * Version: 1.0
 * Author: 瓶子
 */
public class Demo { 
    public static void main(String[] args) {
        ArrayList<Xiaomi> xiaomi = new ArrayList<>();
        xiaomi.add(new Xiaomi());
        ArrayList<Bus> bus = new ArrayList<>();
        bus.add(new Bus());
        
        // run(xiaomi);    // 报错，无法传入
        // run(bus);       // 报错，无法传入
        go(xiaomi);        // 不报错，正常运行
        go(bus);           // 不报错，正常运行
        ArrayList<Dog> dog = new ArrayList<>();
        dog.add(new Dog());
        go(dog);           // 不报错，正常运行，通配符导致方法传入了不该出现的参数。
        raceTrack(xiaomi); // 不报错，正常运行
        raceTrack(bus);    // 不报错，正常运行
        raceTrack(dog);    // 报错，无法传入。方法限制只能Car和其子类进入，Dog不是Car的子类。
        
    }
    // 虽然Xiaomi和Bus都是Car的子类，但是ArrayList<Xiaomi>、ArrayList<Bus>和ArrayList<Car> 之间没有任何关系，不能像
    // 继承那样直接使用父类有关的参数和方法。
    public static void run(ArrayList<Car> cars) {}
    // 使用通配符，可以传入一切类型变量。
    public static void go(ArrayList<?> cars) {}
    // 使用通配符和上下限，限制只有Car的子类可以进入方法
    public static void raceTrack(ArrayList<? extends Car> cars) {}
}
class Car {}
class Bus extends Car {}
class Xiaomi extends Car {}
class Dog{}
```

### 泛型支持的类型
> 泛型只支持对象类型（引用类型），不支持基本数据类型。

泛型如果需要使用基本数据类型，必须在声明泛型时使用基本数据类型的包装类。也就是被包装为对象类型的基本数据。
包装类对应关系如下：

| 基本数据类型 | 对应的包装类 |
| ----------- | ----------- |
| byte | Byte |
| short | Short |
| int | Integer |
| long | Long |
| float | Float |
| double | Double |
| char | Character |
| boolean | Boolean |

使用包装类的集合样例。
```java
public class Demo {
    public static void main( String[] args ) {
        // ArrayList<int> list = new ArrayList<>();        // 报错。
        ArrayList<Integer> list = new ArrayList<>();    // 正确
        list.add( 1 );                // 使用正确的方式添加元素。
        int number = list.get( 1 );   // 使用正确方式获取元素。
    }
}
```

## 三、集合框架
### 认识集合
> 集合是一种容器，用来装数据，类似数组。但是集合的大小可变，比数组更加灵活。

集合分两类，一类是`Collection`，一类是`Map`
* collection：又叫单列集合，集合中每个元素只包含一个值。
* Map：又叫双列集合，集合中每个元素包含两个值，一个键，一个值（键值对）。

### Collection集合
#### Collection集合特点
`Collection`集合下还有两种类别
* `List`系列集合：添加的元素是**有序的、可重复的**。
* * `ArrayList`、`LinekdList`：集合内容有序，可重复，有索引。

* `Set`系列集合：添加的元素是**无序的、不可重复的**。
* * `HashSet`：无序、不重复、无索引
* * `TreeSet`：会对对数据进行排序默认升序排序，不重复，无索引。
* * `LinkedHashSet`：有序、不重复、无索引
* 这三种集合都是`Set`的子接口，他们的特点都是：无序，不可重复。但是每个都有和其他不一样的地方。


#### Collection集合特点展示
```java
/**
 * Collection集合特点展示
 *
 * Version: 1.0
 * Author: 瓶子
 */
public class CollectionDemo {
  public static void main(String[] args) { 
      // List集合展示：有序，可重复，有索引
      List<String> list = new ArrayList<>();
      list.add("hello");
      list.add("world");
      list.add("Java");
      list.add("Java");
      System.out.println(list);     // 输出：[hello, world, Java, Java]    顺序和添加顺序一致。
      String s = list.get(1);       // 输出：world   可以使用索引获取元素。
    
      // Set集合展示：无序，不可重复，无索引
      Set<String> set = new HashSet<>();
      set.add("hello");
      set.add("world");
      set.add("Java");
      set.add("Java");
      System.out.println(set);      // 输出：[Java, hello, world]   无序，不可重复。
      //set.get(1);                 // 不可用，没有get方法
  }
}
```

#### 三种遍历方式的区别

删除流程下：

```java
public class CollectionTraversalDemo {

      Collection<String> carname = new ArrayList<>();
      carname.add("保时捷");
      carname.add("法拉利");

      //1、迭代器--得到这个集合的迭代器对象
      Iterator<String> iterator = carname.iterator();    // 调用迭代器方法
      while (iterator.hasNext()) {          // 判断集合中是否有下一个元素

          System.out.println(next);         // 输出元素
      }
      
      //2、增强for循环--得到集合中的元素
      for (String s : carname) {
          System.out.println(s);
      }
      
      //3、Lambda表达式遍历
      // 匿名函数，Consumer是一个接口式函数，可简化
//      carname.forEach(new Consumer< String >() {
//          @Override
//          public void accept( String s ) {
//              System.out.println(s);
//          }
//      });
      // 简化后。
      carname.forEach(s -> System.out.println(s));

}
```

#### List有序集合的特有功能和原理
List集合在继承了Collection集合的功能后，因为本身是有序集合，他还有自身的独有功能

| 方法名                                   | 描述                              |
|:--------------------------------------|:--------------------------------|
| public void add(int index, E element) | 在指定位置插入元素                       |
| public E remove(int index)            | 删除指定索引处的元素，删除时还会返回这个元素的值        |
| public E set(int index, E element)    | 替换指定索引处的元素，修改时时还会返回这个元素被修改之前的值  |
| public E get(int index)               | 返回指定索引处的元素                      |

如上表格，List集合因为其有序性，额外增加了增删改查四种功能。

##### ArrayList和LinkedList的底层原理
* ArrayList：底层是基于数组存储数据，查询快，增删慢。
* * 查询快（**根据索引查询快**）：数组在JAVA中是连续存储的，查询数据通过指向的地址值和索引定位数，所以查询越快且查询任意数据耗时相同。
* * 增删慢（**插入和删除数据慢**）：ArrayList是可变长度的，增删数据时可能涉及到要把后面很多的数据进行前后移动，所以增删数据慢。

* LinkedList：底层是双链表，查询慢，增删快。
> 链表是一种物理存储单元上非连续、非顺序的存储结构（也数据就是分散在内存中各个地方）。
> 链表由一系列节点（链表结点，link）组成。每个节点由两个部分组成：当前节点数据和下一个数据的地址（单链表）。
> 双链表则是每个节点都有两个指针，一个指针指向前一个节点的地址，一个指针指向下一个节点的地址。
* * 查询慢：单链表每次查询都需要从第一个数据开始查询，一个个寻址到你需要找的数据。
           双链表可以通过寻找数据的相对位置。选择从头或者从尾开始查询，查询速度更快但是占用空间更大。
* * 增删快：增删数据直接更新节点的数据指针，不需要移动数据。特别是双链表增删头尾数据是最快的。
> 基于双链表特性，LinkedList有许多独有的头尾数据操作方法。

#### Set集合的特有功能和原理
> Set集合因为其无序无索引的特性，并没有添加额外的功能。他的功能基本都是继承自Collection集合。

##### 底层原理
* HashSet：底层是哈希表（HashMap）（数组、链表、红黑树），增删改查的性能都相对较好
* * 通过计算数据的哈希值通过算法计算确认存储在数组的一个位置，前后位置不确定（无序）。如果数组同一个位置已数据，且比较后数据不一样，JDK8之前采用链表形式在该位置数据前挂入数据，8之后在后挂入数据。形成链表。
* * * 哈希表`JDK8`之前的构成是`数组+链表`，`JDK8`之后基于`数组+链表+红黑树`（当链表长度超过8且数组长度大于64时，自动将链表转为红黑树）

* TreeSet：底层基于红黑树实现排序，查询慢，增删快。
* * 对于数值类型，默认按数值本身大小排序。
* * 对于字符串类型，默认按字符串字典顺序排序。
* * 对于自定义类型对象，`TreeSet`默认无法排序，需要指定排序规则。无法排序会导致集合报错。
* * * 1、对象类中实现`Comparable`接口，重写`compare`方法，指定大小比较规则。
* * * 2、`TreeSet`集合自带比较器`Comparable`对象，指定比较规则。

* LinkedHashSet：底层是链表+哈希表，查询慢，增删快。
* * 与`HashSet`不同的是，`LinkedHashSet`的每一个数据都增加了一个双链表机制，记录前后数据的位置。虽然数据存储的位置依然是无序的，但是得益于双链表的头尾指针，形成了一种顺序结构。

### Map集合
> Map集合也被叫做“键值对集合”，格式:`{key1=value1,key2=value2,key3=value3,…}`
> Map集合的所有键是不允许重复的，但值可以重复，键和值是一一对应的，每一个键只能找到自己对应的值
#### Map集合体系的特点
* Map集合是双列集合，存储结构为键值对。
* Map系列集合的特点都由键决定。值只是一个附属品，不做要求。
* * HashMap：无序、不重复、无索引
* * LinkedHashMap：有序，不重复，无索引
* * TreeMap：按照大小默认升序排序、不重复、无索引

#### Map系列集合的常用方法
创建集合
* Map<K,V> map = new HashMap<>();
* K，V分别代表键值的泛型。例如：`Map<String,Integer> map = new HashMap<>();`

| 方法名                                                           | 描述                                  |
|:--------------------------------------------------------------|:------------------------------------|
| public V put(K key, V value)                                  | 添加元素                                |
| public V get(Object key)                                      | 根据键获取值                              |
| public V remove(Object key)                                   | 根据键删除元素                             |
| public boolean containsKey(Object key)                        | 判断集合中是否包含指定的键                       |
| public boolean containsValue(Object value)                    | 判断集合中是否包含指定的值                       |
| public Set<K> keySet()                                        | 获取所有键的集合                            |
| public Collection<V> values()                                 | 获取所有值的集合                            |
| public int size()                                             | 获取集合大小                              |
| public void clear()                                           | 清空集合                                |
| public boolean isEmpty()                                      | 判断集合是否为空，返回true，非空false             |
| public Set<Map.Entry<K,V>> entrySet()                         | 获取所有"键值对"然后返回一个set集合，数据类型是Map.Entry |
| default void forEach(Biconsumer<? super K, ? super V> action) | 结合lambda遍历集合|

#### Map集合的遍历方式
* 键找值
* * 先获取`Map`集合所有的键，然后遍历键，根据键找值。

* 键值对（遍历难度较高，需要较深代码理解）
* * 把"键值对"看成一个整体进行遍历。

* Lambda表达式遍历
* * 需要用到`Map`的一个方法：`default void forEach(Biconsumer<? super K, ? super V> action)`，结合`lambda`遍历`Map`集合

```java
import java.util.Map;

public class MapDemo {
    public static void main( String[] args ) {
        Map<String, Integer> map = new HashMap<>();
        map.put("张三", 18);
        map.put("李四", 19);
        map.put("王五", 20);
        
        // 键找值
        Set<String> keys = map.keySet();
        for (String key : keys) {
            Integer value = map.get(key);
            System.out.println(key + "=" + value);
        }
        
        // 键值对遍历
        Set<Map.Entry<String, Integer>> entries = map.entrySet();
        for (Map.Entry<String, Integer> entry : entries) {
            String key = entry.getKey();
            Integer value = entry.getValue();
            System.out.println(key + "=" + value);
        }
        
        // Lambda表达式遍历
//        map.forEach(new BiConsumer<String, Integer>() {
//            @Override
//            public void accept(String key, Integer value) {
//                System.out.println(key + "=" + value);
//            }
//        });
        // 化简后
        map.forEach((key, value) -> System.out.println(key + "=" + value));
        
    }
}            
```

#### Map集合综合样例
```java
public class Demo {
    public static void main(String[] args) {
        /**
         * 组织投票，投票选项分别是A\B\C\D，投票的人有100人，现在需要统计每个选项的投票数
         *
         * param
         * return
         * Version: 1.0
         * Author: 瓶子
         */
        Map<String, Integer> map = new HashMap<>();
```
