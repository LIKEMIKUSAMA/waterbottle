## 一、类的基本概念与定义

### 什么是类？
类是对现实世界中事物的抽象描述，它定义了一类事物的共同属性和行为。例如，"汽车" 类可以包含颜色、品牌、速度等属性，以及启动、加速、刹车等行为。在 Java 中，类是创建对象的模板，而对象是类的具体实例。

### 类的定义语法
定义一个类的基本语法结构如下：
```textmate
[访问修饰符] class 类名 {
    // 成员变量（属性）
    [访问修饰符] [数据类型] 变量名;
    
    // 构造方法
    [访问修饰符] 类名([参数列表]) {
        // 构造方法体
    }
    
    // 成员方法（行为）
    [访问修饰符] [返回类型] 方法名([参数列表]) {
        // 方法体
        return 返回值;
    }
}
```

### 类的命名规范
类名使用驼峰命名法，首字母大写，例如Student、CarManager
类名应尽量使用名词或名词短语，准确描述类的功能
遵循 Java 的包命名规范，例如com.example.demo.User

## 二、类的基本语法详解

### 构造器
> 构造器是创建对象时，对象自动调用的初始化方法，是一种特殊方法，不能写返回值类型，名称必须是类名，就是构造器。
```textmate
    // 构造器
    [访问修饰符] 类名([参数列表]) {
        // 构造方法体
    }
```
构造器又分为有参构造器和无参构造器。
有参构造器指的是构造器定义了参数列表，无参构造器没有参数列表。
```java
public class Student {
    // 成员变量
    private String name;
    private String chinese;
    private String math;
    private String age;

    // 无参构造器
    public Student() {
    }
    // 有参构造器
    public Student( String name, String chinese, String math, String age ) {
        this.name = name;
        this.chinese = chinese;
        this.math = math;
        this.age = age;
    }
    public static void main( String[] args ) {
        Student student1 = new Student();    // 创建对象，调用无参构造器
        Student student2 = new Student("张三", "90", "80", "18");      // 创建对象，调用有参构造器
        
    }
}
```
构造器常见应用场景
多用于创建对象时，同时完成对对象成员变量的初始化赋值。
对象创建时，依据传入参数，我们可以指定对象去调用哪个构造器

**注意事项**
* 构造方法名与类名完全相同，没有返回类型，也不能使用 `void`
* 类默认自带一个无参构造器。当类中没有显式定义构造方法时，编译器会自动生成一个无参构造方法
* 如果为类定义了有参构造器，类默认的无参构造器就消失了。如果还想用无参构造器，必须自己在类里手写出来。
* 创建对象时，如果没有显示调用构造器，那么会自动调用无参构造器。
* 有参构造器可以要求传入全部参数，也可以要求只传入部分参数。即构造方法也是可以和普通方法一样重载的

### this关键字
#### 什么是this关键字
this关键字是一个变量，指的是当前对象，this关键字可以访问当前对象的成员变量和成员方法。

#### this的应用场景
* `this`关键字可以用于解决同名成员变量和成员方法之间的冲突。
* `this`关键字也可以用于调用当前对象的成员方法或构造器。
```java
public class Student {
    private String name;
    private int age;
    
    // 若不使用this，name和age会被视为局部变量
    public Student(String name, int age) {
        this( name);
        this.name = name; // this.name是成员变量，name是参数
        this.age = age;   // this.age是成员变量，age是参数
        System.out.println("重载构造方法被调用");
    }

    public Student(String name){
        this.name = name;
        System.out.println("基础构造方法被调用");
    }
    
    public void setName(String name) {
        this.name = name; // 区分成员变量和方法参数
    }
    
    // 错误示例：没有this会导致成员变量无法被赋值
    public void wrongSetAge(int age) {
        age = age; // 两个age都是方法参数，成员变量未被修改
    }

    // 测试调用
    public static void main(String[] args) {
        User u = new User("admin", "123456", 25);
        // 输出：基础构造方法被调用 -> 重载构造方法被调用
    }
}
```

#### this的使用限制
* 静态方法中不能使用 `this`
  静态方法属于类本身，不与任何对象绑定，而this指向对象，因此静态方法中使用this会报错。
* 构造方法中只能调用一次 `this ()`
  `this([参数])`必须是构造方法的第一行代码，且只能调用一次，避免循环调用。
* 不能在构造方法外调用 `this ()`
  `this()`只能在构造方法内部使用，用于调用其他构造方法。

### 封装
封装是面向对象编程（OOP）的三大特性（封装、继承、多态）之一，其本质是将数据（属性）和操作数据的方法（行为）捆绑在一起，形成一个独立的单元（类），并通过控制访问权限来隐藏内部实现细节，仅对外暴露必要的接口。

#### 封装的的设计要求
> 合理隐藏，合理暴漏。

通常将类的属性设为`private`，通过`public` 修饰的 `getter`/`setter` 方法暴露访问接口

#### 封装的核心目的
* 保护数据的完整性和安全性，避免外部代码对内部状态的非法修改
* 降低代码耦合度，使系统结构更清晰
* 便于维护和扩展，内部实现改变时不影响外部调用
* 封装的属性不会被子类直接访问，但可通过父类的 `getter`/`setter` 间接操作
* 封装的方法可在子类中重写（Override），实现多态性
* 外部调用时无需关心具体子类实现，只通过封装的接口交互

#### 封装的实现
下面是一个封装的例子：
```java
public class Student {
    // 姓名，年龄，身高，性别。全部private修饰隐藏
    private String name;
    private int age;
    private double height;
    private char sex;
    
    // 提供public修饰的getter和setter方法
    public String getName() {
        return name;
    }

    public void setName( String name ) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge( int age ) {
        this.age = age;
    }

    public double getHeight() {
        return height;
    }

    public void setHeight( double height ) {
        this.height = height;
    }

    public char getSex() {
        return sex;
    }

    public void setSex( char sex ) {
        this.sex = sex;
    }
}
```

### 实体类
#### 什么是实体类
实体类是一种特殊类，需要满足两个要求
1. 类的成员变量全部私有，并且提供对应的由 `public`修饰的 `getter`、`setter` 方法。
2. 类中需要提供一个无参构造器，有参构造器可选。

> 我们在封装中提供的代码样例就是一个实体类。这里不再赘述，有兴趣的朋友可以上翻回看。

#### 实体类的应用场景
实体类一般用于封装数据。类的对象只负责数据存取。对数据的业务处理交由其他类的对象完成，实现数据和业务分离。

### static关键字
#### static的作用
`static` 关键字又叫静态修饰关键字，用于修饰成员变量和成员方法，使得这些成员变量和成员方法属于类，而不是对象。
被修饰的成员变量和成员方法叫做：**静态成员变量**和**静态成员方法**。没有被修饰的叫做**实例变量**和**实例方法**

#### static注意事项
* 静态成员变量和静态成员方法在类加载时就被加载，**不需要创建对象就可以访问**，也可以通过对象访问。
* 实例变量和实例方法在创建对象时加载，**必须创建对象才能访问**，它属于每个对象，**只能通过对象访问**。
* 静态方法中可以直接访问静态成员，**不可以**直接访问实例成员。
* 实例方法中既可以直接访问静态成员，**也可以**直接访问实例成员
* 实例方法中可以出现this关键字，静态方法中不可以出现this关键字。

#### static样例
```java
public class Student {
    // 静态变量
    static String name = "张三";
    // 实例变量
    int age;

  public static void main( String[] args ) {
      Student student1 = new Student();
      student1.age = 18;
      
      Student student2 = new Student();
      student2.age = 28;
      
      System.out.println( student1.name + " " + student1.age );     // 输出：张三 18
      System.out.println( student2.name + " " + student2.age );     // 输出：张三 28
      System.out.println( Student.name );                           // 输出：张三
  }
}
```
