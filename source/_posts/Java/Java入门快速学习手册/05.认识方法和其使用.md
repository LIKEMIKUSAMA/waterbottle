~~## 一、Java 方法的核心概念
> 方法是Java里的基石，是Java里最核心的编程概念，Java的一个类是由很多个方法组成，很多个类组成一套处理数据的逻辑关系，很多个逻辑关系最终组成一个完整的Java程序。

Java 方法本质上是类中一段具有特定功能的代码集合，**是完成特定功能的代码块**。相当于一个"黑盒子"，就像家里的咖啡机，接收不同的 “咖啡豆”（参数），经过一系列加工，产出对应的 “咖啡”（返回值）。
```java
// 类比现实：咖啡机就是个"方法"
public Coffee makeCoffee(CoffeeBean bean, int waterML) {
    // 内部实现细节被封装...
    return new Coffee();   // 返回一杯做好的咖啡
}
```
再比如，在一个学生成绩管理系统中，计算学生平均分、查找最高分学生等功能，都可以封装成不同的方法。

✅ 方法的**核心价值**：
1. **代码复用** - 避免重复代码
2. **模块化** - 分解复杂问题
3. **可读性** - 通过方法名理解功能
4. **易维护** - 修改只需调整一处
5. 合理使用方法，能让代码结构清晰，避免重复劳动，提高开发效率。

## 二、Java 方法的构成要素详解
一个完整的 Java 方法由多个部分组成，每个部分都有其独特的作用，下面我们逐一深入分析。
```java
// 方法的构成主要由下面部分组成
[访问修饰符] [static/final] 返回类型 方法名(参数列表) [throws 异常] {
    // 方法体
    return 返回值; // void方法可省略
}
```
### 1、访问修饰符：方法的 “权限与属性标签”
> 修饰符决定了方法的访问权限和特性。
表格如下

| 修饰符      | 作用域                     | 常见场景               |
|-------------|---------------------------|------------------------|
| `public`    | 所有类可见                | 工具类方法             |
| `protected` | 同包+子类可见             | 父类扩展方法           |
| `默认`      | 同包可见                  | 内部工具方法           |
| `private`   | 仅当前类可见              | 辅助方法/封装细节      |

#### public
> `public` 修饰符表示方法是公共的，就像公共场所，任何类都可以访问这个方法。比如在一个工具类中，用于数据加密的encryptData方法，希望其他类都能使用，就可以用public修饰。
```java
   public class Utility {
       public static String encryptData(String data) {
           // 加密逻辑
           return encryptedData;
       }
   }
``` 

#### private
> `private` 修饰符意味着方法是私有的，如同私人房间，只有在本类中才能访问。当一个方法只是辅助本类其他方法完成功能，不希望被外部调用时，就可以使用private。

例如在一个用户类中，用于校验密码格式的validatePasswordFormat方法：
```java
class User {
    private boolean validatePasswordFormat(String password) {
        // 密码格式校验逻辑
        return isValid;
    }
}
```


#### static
> `static` 修饰符表示静态方法，静态方法可以直接通过类名调用，无需创建类的实例。

> 静态方法可以访问静态成员变量和静态方法，但不能访问实例成员变量和实例方法。

比如数学工具类中的calculateCircleArea方法计算圆的面积：
```java
public class MathUtils {
   public static double calculateCircleArea(double radius) {
        return Math.PI * radius * radius;
    }
}
// 调用方式
double area = MathUtils.calculateCircleArea(5);

```

还有protected（受保护的，用于继承场景）、final（方法不能被重写）等修饰符，它们在不同场景下发挥着重要作用。

### 返回类型：方法的 “产出物类型”
> 返回值类型明确了方法执行结束后返回的数据类型。如果方法没有返回值，就用void表示，比如打印日志的logMessage方法：
```java
public void logMessage(String message) {
    System.out.println("日志信息：" + message);
}
```

> 若有返回值，必须与定义的类型一致。例如计算两个整数乘积的multiply方法：
```java
public int multiply(int num1, int num2) {
    return num1 * num2;
}
```

方法返回类型详细样例如下
```java
/**
* 方法返回类型样例
* 
* Version: 1.0
* Author: 瓶子
*/
// 返回基本类型
public int add(int a, int b) {
    return a + b;
}

// 返回对象类型
public String getUserName(User user) {
    return user.getName();
}
2
// 无返回值 (void)
public void printLog(String message) {
    System.out.println("[LOG] " + message);
}

// 返回集合
public List<Integer> getEvenNumbers(List<Integer> numbers) {
    return numbers.stream()
                  .filter(n -> n % 2 == 0)
                  .collect(Collectors.toList());
}
```
> 值得注意的是，return语句不仅用于返回值，还能提前结束方法执行，即使方法是void类型。

### 方法名
方法名，顾名思义，是方法的专属名字，它唯一标识了该方法。

方法名要遵循驼峰命名法，且能准确反映方法功能。例如，将字符串转换为大写的convertToUpperCase方法，从名字就能清晰知道它的用途。避免使用模糊、无意义的名称，否则会给代码阅读和维护带来困难。

### 参数列表
> 参数列表是方法执行时所需的数据。
 
参数可以是基本数据类型，如int、double，也可以是引用数据类型，如String、自定义类。方法可以有多个参数，也可以没有参数。

例如，计算长方形面积的calculateRectangleArea方法接收长和宽两个参数：
```java
public double calculateRectangleArea(double length, double width) {  //length 和 width 是就是参数，double 是参数的类型
    return length * width;
}
```

当方法需要处理多个同类型数据时，还可以使用数组作为参数

比如计算数组中所有元素和的sumArrayElements方法：
```java
public int sumArrayElements(int[] array) {   //array 是一个int类型的数组
    int sum = 0;
    for (int num : array) {
        sum += num;
    }
    return sum;
}
```


### 方法体
方法体是实现具体功能的代码块，用`{}`包裹。在方法体中，根据业务逻辑编写代码，可能包含变量声明、条件判断、循环等操作。

例如，判断一个数是否为质数的isPrime方法：
```java
/**
* 判断一个数是否为质数
* 
* Version: 1.0
* Author: 瓶子
*/
public boolean isPrime(int number) {  //  方法代码块的左括号
    if (number < 2) {
      return false;
    }
    for (int i = 2; i <= Math.sqrt(number); i++) {
        if (number % i == 0) {
            return false;
        }
    }
    return true;
}  //方法代码块的右括号，也是结束括号
```



## 三、Java 方法的多样使用场景

### 无参数无返回值方法
> 这类方法常用于执行一些简单的、不需要输入输出的操作，比如初始化系统配置、播放提示音等。

以打印系统启动信息为例：
```java
/**
* 模拟系统启动
* 
* Version: 1.0
* Author: 瓶子
*/
public class SystemInitializer { 
    public static void printStartupMessage() {
        System.out.println("系统正在启动...");
        System.out.println("加载配置文件...");
        System.out.println("启动完成！");
    }
}
   // 在主程序中调用
public class Main { 
    public static void main(String[] args) {
        SystemInitializer.printStartupMessage();
    }
}
```


### 有参数无返回值方法
> 当方法需要根据外部传入的数据进行处理，但不需要返回结果时适用。

比如，向文件中写入日志信息，需要传入日志内容：
```java
/**
* 日志记录方法样例
* 
* Version: 1.0
* Author: 瓶子
*/
import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class LogWriter {
    public static void writeLog(String logMessage) {
        try (BufferedWriter writer = new BufferedWriter(new FileWriter("log.txt", true))) {
            writer.write(logMessage);
            writer.newLine();
        } 
        catch (IOException e) {
            e.printStackTrace();
        }
    }
}
// 调用示例
public class Main {
    public static void main(String[] args) {
    LogWriter.writeLog("用户登录成功");
    }
}
```


### 有参数有返回值方法
> 这是最常用且的方法类型，用于处理数据并返回结果。例如，从数据库中查询用户信息，传入用户 ID，返回对应的用户对象：
```java
/**
* 有参数有返回值方法样例
* 
* Version: 1.0
* Author: 瓶子
*/
class User { 
    private String id;
    private String name;
    private int age;

    // 省略构造函数、getter和setter方法

    public static User getUserById(String userId) {
    // 模拟从数据库查询操作
       if ("123".equals(userId)) {
           return new User("123", "张三", 25);
       }
       return null;
    }
}
   // 调用示例
public class Main {
    public static void main(String[] args) {
        User user = User.getUserById("123");
        if (user != null) {
            System.out.println("用户姓名：" + user.getName());
        }
    }
}
```

### 方法的重载
> 在同一个类中，允许定义多个方法名相同但参数列表不同的方法，这就是方法重载。
> 编译器会根据调用时传入的**参数类型和个数自动匹配**对应的方法。

例如，计算图形面积的方法，针对不同图形可以进行重载：

```java
/**
* 方法的重载样例
* 
* Version: 1.0
* Author: 瓶子
*/
public class AreaCalculator {
    public static double calculateArea(double radius) {
        // 计算圆形面积
        return Math.PI * radius * radius;
    }

    public static double calculateArea(double length, double width) {
        // 计算长方形面积
        return length * width;
    }

    public static double calculateArea(double base, double height) {
        // 计算三角形面积
        return 0.5 * base * height;
    }
}
   // 调用示例
public class Main {
    public static void main(String[] args) {
        double circleArea = AreaCalculator.calculateArea(5);
        double rectangleArea = AreaCalculator.calculateArea(4, 6);
        double triangleArea = AreaCalculator.calculateArea(3, 8);
        System.out.println("圆形面积：" + circleArea);
        System.out.println("长方形面积：" + rectangleArea);
        System.out.println("三角形面积：" + triangleArea);
    }
}

```

### 递归方法
> 递归方法是指**在方法体内调用自身的方法**。
> 常用于解决可以分解为相似子问题的情况，如计算阶乘、遍历树形结构等。

以计算阶乘为例 ：
```java
/**
* 方法的递归使用样例，计算阶乘
* 
* Version: 1.0
* Author: 瓶子
*/
public class FactorialCalculator {
    public static int calculateFactorial(int n) {
        if (n == 0 || n == 1) {
            return 1;
        }
        return n * calculateFactorial(n - 1);
    }
}
   // 调用示例
public class Main {
    public static void main(String[] args) {
        int result = FactorialCalculator.calculateFactorial(5);
        System.out.println("5的阶乘是：" + result);
    }
}
```
> **注意：** 使用递归方法时要注意设置终止条件，否则会导致栈溢出错误。
### 可变参数方法
> 在 `Java 5` 之后，引入了可变参数，允许方法接收任意数量的同类型参数。可变参数**本质上是一个数组**，在方法内部可以像操作数组一样处理参数。

例如，计算多个整数的和：
```java
public class SumCalculator {
    public static int sum(int... numbers) {  // 使用三个"."来表示这是一个可变参数
        int total = 0;
        for (int num : numbers) {
            total += num;
        }
        return total;
    }
}
   // 调用示例
public class Main {
    public static void main(String[] args) {
        int sum1 = SumCalculator.sum(1, 2, 3);
        int sum2 = SumCalculator.sum(4, 5, 6, 7, 8);
        System.out.println("和1：" + sum1);
        System.out.println("和2：" + sum2);
    }
}

``` 
  
## 四、方法使用的注意事项与黄金法则
### 使用注意事项
方法的作用域：方法中定义的变量是局部变量，只在方法体内有效，方法执行结束后，局部变量就会被销毁。
方法调用的性能：过多的方法调用会产生一定的性能开销，尤其是在循环中频繁调用方法时，要考虑性能优化。
方法的注释：为了提高代码的可读性和可维护性，要给方法添加清晰的注释，说明方法的功能、参数含义、返回值意义等。例如：
```java
/**
* 计算两个整数的和
*
* @param num1 第一个整数
* @param num2 第二个整数
* @return 两个整数的和
* Version: 1.0
* Author: 瓶子
*/

public int add(int num1, int num2) {
    return num1 + num2;
}

```

### 方法的黄金法则
1. **单一职责**：一个方法只做一件事
2. **见名知意**：方法名精确反映功能
3. **参数精简**：不超过3个参数（多用对象封装）
4. **防御编程**：对输入参数进行校验
5. **避免副作用**：不修改非局部变量（纯函数更安全）
6. **长度控制**：方法体不超过屏幕一屏（约30行）
7. **异常处理**：合理使用throws或try-catch
