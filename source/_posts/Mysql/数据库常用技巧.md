## 一、配置类

### 1.1、修改连接数

```bash
# my.cnf中添加配置项
# 最大连接数
max_connections=1000
```

---

### 1.2、只读模式强制启动

```bash
# my.cnf 添加
innodb_force_recovery = 6
```

---

### 1.3、表名大小写区分

配置方法一：数据内修改配置变量

```sql
-- 查看 lower_case_table_names 变量的值
SHOW VARIABLES LIKE 'lower_case_table_names';

-- 设置 lower_case_table_names 变量的值为 1（需要重启 MySQL 服务）
SET GLOBAL lower_case_table_names = 1;
```

配置方法二：my.cnf中添加配置项

```bash
# 在【mysqld】下增加lower_case_table_names=1
lower_case_table_names=1
```

---

### 1.4、开启通用日志

```bash
# 在my.cnf文件中的`[mysqld]`模块下添加添加
# 设置日志使用系统时间
log_timestamps = SYSTEM 
general_log = 1
general_log_file = /var/log/mysql/mysql.log
```

---

### 1.5、解除用户访问限制

```sql
select * from `mysql`.`user` where user = 'root';
-- 如果不是全权限或者指定IP，看情况执行下面的SQL
update `mysql`.`user` set host = '%' where user = 'root';
-- 最后执行，刷新
flush privileges
```

---

### 1.6、释放数据库连接

```bash
# 30分钟无请求则释放连接，默认值为28800即8个小时。
wait_timeout=1800
```

---

### 1.7、提示数据库大量连接错误无法连接

报错：`Host is blocked because of many connection errors;unblock with 'mysqladmin flush-hosts'`
服务器连上数据库后，执行：`FLUSH HOSTS`;

---

### 1.8、修改数据库缓冲池

```bash
# 调整为4G
innodb_buffer_pool_size = 4G
```


## 二、增删改查SQL

### 2.1、查询数据库内表数据大小

```sql
-- 查询数据库表行数
SELECT TABLE_NAME,TABLE_ROWS FROM information_schema.`TABLES`
WHERE TABLE_SCHEMA = (SELECT database()) ORDER BY TABLE_ROWS DESC;
```

---

### 2.4、筛选重复字段

```sql
select *,count(pkid) from t_temp_meter_YH0105_copy GROUP BY pkid having count(pkid)>1
```

---

### 2.5、筛选字段大于指定长度的结果

```sql
select * from t_temp_gas_user_YH0105_copy where length(usertype)>20
```

---

### 2.6、按指定规则排序

```sql
-- 指定规则，如1>3>2>0，order by后使用FIELD来指定所需字段的排序规则
SELECT `user` FROM user WHERE `user_status` in (0,1,2,3) ORDER BY FIELD(`user_status`,1,0,2,3);
```

---

### 2.7、时间格式化

```bash
# 使用函数DATE_FORMAT()
DATE_FORMAT(date,"format")
```

format参数的常用格式有
| 格式符 | 描述                    | 格式符 | 描述 |
| --- | ---                       |--- | --- |
| `%y` | 年，2 位                  |`%Y` | 年，4 位 |
| `%m` | 月，数值(00-12)           | `%d` | 月的天，数值(00-31) |
| `%H` | 小时 (00-23)              |`%h` | 小时 (01-12) |
| `%i` | 分钟，数值(00-59)         | `%S` | 秒(00-59) |
| `%T` | 时间， 24-小时 (hh:mm:ss) | `%r` | 时间，12-小时（hh:mm:ss AM 或 PM） |

---

### 2.8、SQL执行时长检查（慢查询）

```sql
-- 任选其一执行
SHOW FULL PROCESSLIST;
select * from PROCESSLIST order by time DESC limit 10;

-- 找到执行时间过长导致慢查询的SQL的id。
kill id;
```

---

### 2.9、备份

```bash
# 格式：mysqldump -h主机IP -P端口 -u用户名 -p密码 --database 数据库名 > 文件名.sql
# 本地备份可以不添加端口和主机IP，username、passward是数据库用户名和密码
mysqldump -h *.*.*.* -p 3306 -u username -p password --database mysql > /data/backup/mysql.sql
# 导出的数据有可能比较大，不好备份到远程，这时候就需要进行压缩
mysqldump -h *.*.*.* -p 3306 -u username -p password --database mysql | gzip > /data/backup/mysql.sql.gz
```
