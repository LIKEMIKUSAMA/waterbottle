Java进阶01.异常、泛型和集合框架

本文最后更新于 2025年11月10日 上午

本章主要学习异常、泛型和集合框架

一、异常

异常是程序运行时出现的错误，导致程序无法继续运行的情况。

异常的分类

  • 运行时异常（RuntimeException）
    编译阶段不会出现错误提醒，运行时出现的异常(如:数组索引越界异常)编译时异常:
    注意：运行时异常很可能会导致程序中断！

例如大家熟悉的空指针异常基本都是运行时异常。

  • 编译时异常（Exception）
    编译阶段就会出现错误提醒的，且编译阶段不可通过。(如:日期解析异常)

出现代码编辑器的错误提示基本都是编译时异常。主要为了提醒编码者这里代码有问题。

异常的处理

  1. 抛出异常：又叫找背锅侠，就是把异常抛给调用者处理。
  2. 捕获异常：自己捕获异常，自己处理。

异常的作用

1、用来定位程序BUG的关键信息。

例如上面的例子，当字符串为NULL时，会抛出Exception异常，并执行catch块中的代码。

2、作为方法的特殊返回，以便通知上层调用者需要进行什么处理。

当方法执行成功时，返回一个结果，当方法执行失败时，返回一个异常。便于定位BUG

二、泛型

认识泛型

定义类、接口、方法时，同时声明了一个或者多个类型变量（如<E>）
称为泛型类，泛型接口，泛型方法，统称泛型。

作用：泛型提供了在编译阶段约束集合元素类型的能力，并进行自动检查的能力。可以避免强制类型转换，及可能出现的异常。
本质：泛型本质上是把具体的数据类型作为参数传递给类型变量。

泛型类

声明时定义了泛型的类就是泛型类，声明类型变量时，类型变量名称必须大写。

泛型接口

声明接口时时定义了泛型的接口就是泛型接口，声明类型变量时，类型变量名称必须大写。

泛型方法

声明方法时时定义了泛型的方法就是泛型方法，声明类型变量时，类型变量名称必须大写。

泛型样例

泛型通配符与上下限。

泛型通配符：?，可以在“使用泛型”的时候表示一切泛型；ETKV是在定义泛型的时候使用。

通配符一般配合上下限一起使用

泛型通配符的上限：<? extends T>，表示该泛型只能是T类或者T的子类；
泛型通配符的下限：<? super T>，表示该泛型只能是T类或者T的父类。

泛型支持的类型

泛型只支持对象类型（引用类型），不支持基本数据类型。

泛型如果需要使用基本数据类型，必须在声明泛型时使用基本数据类型的包装类。也就是被包装为对象类型的基本数据。
包装类对应关系如下：

基本数据类型 对应的包装类
byte Byte
short Short
int Integer
long Long
float Float
double Double
char Character
boolean Boolean

三、集合框架

认识集合

集合是一种容器，用来装数据，类似数组。但是集合的大小可变，比数组更加灵活。

集合分两类，一类是Collection，一类是Map

  • collection：又叫单列集合，集合中每个元素只包含一个值。
  • Map：又叫双列集合，集合中每个元素包含两个值，一个键，一个值（键值对）。

Collection集合

Collection集合特点

Collection集合下还有两种类别

  • List系列集合：添加的元素是有序的、可重复的。

    • ArrayList、LinekdList：集合内容有序，可重复，有索引。

  • Set系列集合：添加的元素是无序的、不可重复的。

    • HashSet：无序、不重复、无索引
    • TreeSet：会对对数据进行排序默认升序排序，不重复，无索引。
    • LinkedHashSet：有序、不重复、无索引

  • 这三种集合都是Set的子接口，他们的特点都是：无序，不可重复。但是每个都有和其他不一样的地方。

List有序集合的特有功能和原理

ArrayList和LinkedList的底层原理

  • ArrayList：底层是基于数组存储数据，查询快，增删慢。

    • 查询快（根据索引查询快）：数组在JAVA中是连续存储的，查询数据通过指向的地址值和索引定位数，所以查询越快且查询任意数据耗时相同。
    • 增删慢（插入和删除数据慢）：ArrayList是可变长度的，增删数据时可能涉及到要把后面很多的数据进行前后移动，所以增删数据慢。

  • LinkedList：底层是双链表，查询慢，增删快。

链表是一种物理存储单元上非连续、非顺序的存储结构（也数据就是分散在内存中各个地方）。
链表由一系列节点（链表结点，link）组成。每个节点由两个部分组成：当前节点数据和下一个数据的地址（单链表）。
双链表则是每个节点都有两个指针，一个指针指向前一个节点的地址，一个指针指向下一个节点的地址。

    • 查询慢：单链表每次查询都需要从第一个数据开始查询，一个个寻址到你需要找的数据。
      双链表可以通过寻找数据的相对位置。选择从头或者从尾开始查询，查询速度更快但是占用空间更大。
    • 增删快：增删数据直接更新节点的数据指针，不需要移动数据。特别是双链表增删头尾数据是最快的。

基于双链表特性，LinkedList有许多独有的头尾数据操作方法。

Set集合的特有功能和原理

Set集合因为其无序无索引的特性，并没有添加额外的功能。他的功能基本都是继承自Collection集合。

底层原理
  • HashSet：底层是哈希表（HashMap）（数组、链表、红黑树），增删改查的性能都相对较好

JDK8之前，哈希表=数组+链表，JDK8之后改为了数组+链表+红黑树。

    • 通过计算数据的哈希值通过算法计算确认存储在数组的一个位置，前后位置不确定（无序）。如果数组同一个位置已数据，且比较后数据不一样，JDK8之前采用链表形式在该位置数据前挂入数据，8之后在后挂入数据。形成链表。
      • 哈希表JDK8之前的构成是数组+链表，JDK8之后基于数组+链表+红黑树（当链表长度超过8且数组长度大于64时，自动将链表转为红黑树）

  • TreeSet：底层基于红黑树实现排序，查询慢，增删快。

    • 对于数值类型，默认按数值本身大小排序。
    • 对于字符串类型，默认按字符串字典顺序排序。
    • 对于自定义类型对象，TreeSet默认无法排序，需要指定排序规则。无法排序会导致集合报错。
      • 1、对象类中实现Comparable接口，重写compare方法，指定大小比较规则。
      • 2、TreeSet集合自带比较器Comparable对象，指定比较规则。

  • LinkedHashSet：底层是链表+哈希表，查询慢，增删快。

    • 与HashSet不同的是，LinkedHashSet的每一个数据都增加了一个双链表机制，记录前后数据的位置。虽然数据存储的位置依然是无序的，但是得益于双链表的头尾指针，形成了一种顺序结构。

Map集合

Map集合也被叫做“键值对集合”，格式:{key1=value1,key2=value2,key3=value3,…}
Map集合的所有键是不允许重复的，但值可以重复，键和值是一一对应的，每一个键只能找到自己对应的值

Map集合体系的特点

  • Map集合是双列集合，存储结构为键值对。
  • Map系列集合的特点都由键决定。值只是一个附属品，不做要求。
    • HashMap：无序、不重复、无索引
    • LinkedHashMap：有序，不重复，无索引
    • TreeMap：按照大小默认升序排序、不重复、无索引

Map系列集合的常用方法

  • Map<K,V> map = new HashMap<>();
  • K，V分别代表键值的泛型。例如：Map<String,Integer> map = new HashMap<>();

public V put(K key, V value) 添加元素
public V get(Object key) 根据键获取值
public V remove(Object key) 根据键删除元素
public boolean containsKey(Object key) 判断集合中是否包含指定的键
public boolean containsValue(Object value) 判断集合中是否包含指定的值
public Set keySet() 获取所有键的集合
public Collection values() 获取所有值的集合

public void clear() 清空集合

public Set<Map.Entry<K,V>> entrySet() 获取所有"键值对"然后返回一个set集合，数据类型是Map.Entry

Map集合的遍历方式

  • 键找值

    • 先获取Map集合所有的键，然后遍历键，根据键找值。

  • 键值对（遍历难度较高，需要较深代码理解）

    • 把"键值对"看成一个整体进行遍历。

  • Lambda表达式遍历

    • 需要用到Map的一个方法：default void forEach(Biconsumer<? super K, ? super V> action)，结合lambda遍历Map集合

Map集合综合样例

底层实现

  • HashMap

    • 底层与HashSet集合底层原理一致，基于哈希表实现。只是Set集合中的元素只要键数据，不需要值数据而已

  • LinkedHashMap

    • 底层原理基于LinkedHashSet
    • 底层数据结构基于哈希表实现，只是每个键值对元素额外多了一个双链表机制记录元素顺序(保证有序)

  • TreeMap

    • 底层与TreeSet集合底层原理一只，基于红黑树实现排序。

Stream流

JDK8开始新增的API，属于java.util.stream，可用于操作集合或者数组的数据。

  • 优势：
    • 大量结合了Lambda的语法风格来编程，功能强大，性能高效，代码简洁，可读性好。

获取Stream流

获取集合的Stream流

Collection提供的方法 说明
default Stream stream() 返回此集合的Stream流

获取数组的Stream流

Arrays类提供的方法 说明
static Stream stream(T[] array) 返回指定数组的Stream流
Stream类提供的方法 说明
static Stream of(T… values) 返回一个包含多个元素的Stream流

Stream流的中间方法

Stream流提供的中间方法，指的是调用完成后会返回新的Stream流，方便继续使用（支持链式编程）

Stream流中间方法 说明
Stream filter(Predicate<? super T> predicate) 对流中的数据进行过滤
Stream sorted() 对流中的元素进行升序排序
Stream sorted(Comparator<? super T> comparator) 按制定规则进行排序
Stream limit(long maxSize) 获取前几个元素
Stream skip(long n) 跳过前几个元素
Stream distinct() 获取去重后的数据
Stream map(Function<? super T, ? extends R> mapper) 对元素加工，并返回对应的新流（映射加工方法）
static Stream concat(Stream a,Stream b) 合并两个流为一个流，被合并的流必须是合并后的流的子类

Stream流的终结方法

终结方法指：调用完成后，不会返回新的Stream，没办法继续使用流进行链式编程了，是最终的方法。

Stream流终结方法 说明
void forEach(Consumer<? super T> action) 对流运算后的元素执行遍历
long count() 获取流中元素个数
Optional max(Comparator<? super T> comparator) 使用比较器，获取流中的最大值
Optional min(Comparator<? super T> comparator) 使用比较器，获取流中的最小值

收集Stream流

把Stream流操作后的记过转回到集合或数组中去
Stream流只是数据的简便操作工具，集合或者数组才是数据的存储容器，是我们需要的东西。

Stream提供的常用收集方法 说明
collect(Collector collector) 把Stream流操作后的结果转回到集合中去
Object toArray() 把Stream流操作后的结果转回到数组中去
Collectors工具类提供的收集方式 说明
public static Collection toList() 把Stream流操作后的记过转回到list集合中去
public static Collection toSet() 把Stream流操作后的记过转回到set集合中去
public static Collection toMap(Function keyMapper,Function valueMapper) 把元素收集到Map集合中

注意 Stream流只能使用一次，使用后，会关闭，不能再次使用。想要使用需要建一个新的Stream流对象。
